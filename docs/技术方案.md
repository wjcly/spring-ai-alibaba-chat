### 2.2 向量化实现

```java
// 使用阿里云百炼 Embedding 模型
public float[] embedDocument(String content) {
    List<Float> embedding = embeddingModel.embed(content);
    float[] vector = new float[embedding.size()];
    for (int i = 0; i < embedding.size(); i++) {
        vector[i] = embedding.get(i);
    }
    return vector;
}

// 余弦相似度计算
public double cosineSimilarity(float[] v1, float[] v2) {
    double dotProduct = 0.0;
    double norm1 = 0.0;
    double norm2 = 0.0;
    for (int i = 0; i < v1.length; i++) {
        dotProduct += v1[i] * v2[i];
        norm1 += v1[i] * v1[i];
        norm2 += v2[i] * v2[i];
    }
    return dotProduct / (Math.sqrt(norm1) * Math.sqrt(norm2));
}
```

### 2.3 检索优化策略

1. **分块策略**: 文档按 500 字符分块，重叠 50 字符
2. **阈值过滤**: 相似度低于 0.7 的文档过滤
3. **场景过滤**: 按工地安全/工业规范分类检索
4. **Top N 限制**: 最多返回 5 个最相关文档

## 3. 工程化设计

### 3.1 限流配置

```java
@Bean
public RateLimiter rateLimiter() {
    RateLimiterConfig config = RateLimiterConfig.custom()
        .limitRefreshPeriod(Duration.ofSeconds(1))
        .limitForPeriod(10)  // 每秒 10 个请求
        .timeoutDuration(Duration.ofMillis(500))
        .build();
    return RateLimiterRegistry.of(config).rateLimiter("ai-service");
}
```

### 3.2 重试配置

```java
@Bean
public Retry retry() {
    RetryConfig config = RetryConfig.custom()
        .maxAttempts(3)  // 最多重试 3 次
        .waitDuration(Duration.ofMillis(1000))
        .retryOnException(t -> !(t instanceof IllegalArgumentException))
        .build();
    return RetryRegistry.of(config).retry("ai-service");
}
```

### 3.3 熔断配置

```java
@Bean
public CircuitBreakerRegistry circuitBreakerRegistry() {
    CircuitBreakerConfig config = CircuitBreakerConfig.custom()
        .failureRateThreshold(50)  // 失败率 50% 触发
        .waitDurationInOpenState(Duration.ofSeconds(30))
        .slidingWindowSize(10)
        .minimumNumberOfCalls(5)
        .build();
    return CircuitBreakerRegistry.of(config);
}
```

### 3.4 日志埋点

```java
@Around("aiServiceMethods()")
public Object logAiService(ProceedingJoinPoint joinPoint) throws Throwable {
    long startTime = System.currentTimeMillis();
    logger.info("AI 服务调用开始 - {}.{}", className, methodName);
    
    try {
        Object result = joinPoint.proceed();
        long duration = System.currentTimeMillis() - startTime;
        logger.info("AI 服务调用完成 - 耗时：{}ms", duration);
        return result;
    } catch (Exception e) {
        logger.error("AI 服务调用失败 - 耗时：{}ms - 错误：{}", duration, e.getMessage());
        throw e;
    }
}
```

## 4. 工业场景落地

### 4.1 工地安全场景

**需求分析**:
- 工人安全意识薄弱，需要了解安全操作规程
- 安全检查频繁，需要快速生成标准化报告
- 事故案例需要总结学习

**解决方案**:
1. 建设安全知识库 (规范 + 流程 + 案例)
2. 提供智能问答 (工人随时咨询)
3. 自动生成报告 (检查报告/整改通知)
4. 培训材料生成 (安全课件)

**部署架构**:
```
工地现场终端 → 边缘网关 → 云端 AI 服务
     │
     └─→ 本地缓存 (离线问答)
```

### 4.2 工业规范场景

**需求分析**:
- 规范文档繁多，查询困难
- 合规性检查耗时
- 应急处置需要快速响应

**解决方案**:
1. 规范文档向量化存储
2. 智能检索 + AI 解读
3. 应急预案自动生成
4. 合规性自动检查

## 6. 安全考虑

1. **API Key 保护**: 使用环境变量或密钥管理系统
2. **数据脱敏**: 日志中脱敏敏感信息
3. **访问控制**: 集成 OAuth2/JWT 认证
4. **审计日志**: 记录所有 AI 调用日志
5. **内容审核**: 对 AI 生成内容进行审核
